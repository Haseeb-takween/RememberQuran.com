**Status**: Implemented

## Context

Milestone 1 shipped the reading experience. Milestone 2 adds listening: reciter audio (2 launch voices, registry ready for 20+), playback controls (per-ayah, continuous surah, repeat for memorisation, speed), a persistent mini player, real-time word-by-word highlight sync, click-to-hear word audio, and Quran Radio.

M1 deliberately left hooks for this: `ArabicWord` had stubbed `isHighlighted` / word-click props, `Word.audio_url` was already fetched, disabled Play buttons sat in `ReaderControls` / `AyahBlock`, and `[surahId]/layout.tsx` reserved a `data-slot="audio-dock"` slot. M2 wired all of that without changing the reader-settings schema or the Quran text API layer.

**Hard constraints (held throughout build):**

1. **Zero regression on M1** — strictly additive; when no audio plays, the app renders exactly as before.
2. **Best- and worst-case handling** — every feature degrades gracefully (network failures, missing/malformed timing data, iOS restrictions).

## Verified data sources

Full detail in `[docs/audio-sources.md](audio-sources.md)`. Summary as used in code:

**Primary — QDC chapter-audio endpoint** (same open API family as verse text):

```
GET https://api.qurancdn.com/api/qdc/audio/reciters/{recitationId}/audio_files?chapter={n}&segments=true
```

- One MP3 per surah + `verse_timings` (per-verse `timestamp_from` / `timestamp_to` and word `segments`).
- Gapless continuous playback; prev/next/repeat are `currentTime` seeks; timings arrive in the same response.
- **Segment tuple = `[wordPosition, startMs, endMs]`**, 1-based, matching `Word.position` for `char_type_name === "word"`. Timestamps are absolute within the chapter file.
- **Launch reciters:** Alafasy **7**, Sudais **3** — both have word segments.

**Rejected alternative:** `v4/recitations/{id}/by_chapter/{n}` — per-ayah files, no timings, src-swap gaps, 286 requests for Al-Baqarah.

**Word-level audio:** `Word.audio_url` is relative (`wbw/001_001_001.mp3`) → prefix `https://audio.qurancdn.com/`. End markers carry `null`.

**Data-quality gotchas** (handled in `src/lib/wordSync.ts`): malformed segment entries (Alafasy 1:3 interleaves single-element arrays); Sudais float timestamps; a verse's first segment can start before its `timestamp_from` (resolve verse first, then word); gaps between segments keep the previous word highlighted to avoid flicker.

## Architecture (as implemented)

**React Context + a small external store — no Zustand** (matches M1's context-based prefs).


| Piece                   | Path                                   | Role                                                                                                                                  |
| ----------------------- | -------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| Types                   | `src/types/audio.ts`                   | Reciter, repeat config, playback speed union                                                                                          |
| Reciter registry        | `src/lib/audioSources.ts`              | `RECITERS`, `getReciter`, `getWordAudioUrl`                                                                                           |
| QDC fetch               | `src/lib/audioApi.ts`                  | Chapter audio + timings; promise cache; eviction on failure for retry                                                                 |
| Sync math               | `src/lib/wordSync.ts`                  | `sanitizeTimings`, `findVerseIndex`, `findWordPosition`                                                                               |
| High-frequency position | `src/lib/playbackStore.ts`             | `{ verseKey, wordPosition, timeMs }` via rAF; `useSyncExternalStore` selectors so only the active verse re-renders                    |
| Player state            | `src/context/AudioPlayerContext.tsx`   | Singleton hidden `<audio>`; `useReducer` for status/mode/repeat/error; separate stable **actions** context for `ArabicWord` consumers |
| UI                      | `src/components/audio/*`               | Bar, dock spacer, play button, reciter/speed/repeat, radio panel                                                                      |
| Persistence             | `localStorage` key `rq-audio-settings` | `{ reciterId, speed }` only — `rq-reader-settings` never touched                                                                      |


**Playback model:** one main `<audio>` for recitation; lazy secondary `Audio()` for word clips (pauses main, resumes on end/error). All playback starts from a user gesture; radio surah advance reuses the unlocked element inside `ended`. Safari gets `playbackRate` re-applied on every `loadedmetadata`.

**Mount points:** `AudioPlayerProvider` in `providers.tsx` (inside `ReaderSettingsProvider`); `AudioPlayerBar` in root `layout.tsx`; `AudioDockSpacer` replaces the M1 dock div in `[surahId]/layout.tsx` (`h-0` idle → `h-16` while playing).

## Build phases (plan → code)

Each phase was gated on: `tsc` clean → `next build` clean → M1 regression spot-check (word tooltip/popover, display modes, settings persistence).

### Phase 1 — Data layer

**Files:** `src/types/audio.ts`, `src/lib/audioSources.ts`, `src/lib/audioApi.ts`, `src/lib/wordSync.ts`

Reciter registry, QDC fetch with cache, pure sync helpers (binary search, sanitizer). Verified live against chapter 1 for both reciters; malformed Alafasy segments dropped; verse-first resolution at overlapping boundaries.

### Phase 2 — Player core

**Files:** `src/lib/playbackStore.ts`, `src/context/AudioPlayerContext.tsx`, `src/components/providers.tsx`

Load flow with stale-token guard (rapid reciter/chapter switches), `safePlay` for AbortError/NotAllowedError, rAF sync loop, repeat/radio boundary handling, pending seek on `loadedmetadata`, pause/error guards when `stop()` clears src.

### Phase 3 — Reader playback wiring

**Files:** `PlayAyahButton.tsx`; updates to `AyahBlock`, `ArabicLine`, `ArabicWord`, `ReadingModeView` (`ReadingVerse` per-verse subscription), `WordMeaningContent` (speaker button), `ReaderControls` (play-surah)

M1 play placeholders → real controls. `ArabicWord` wires `isHighlighted`; desktop click/Enter/Space → `playWord`; touch tap → popover unchanged, speaker inside for pronunciation.

### Phase 4 — Mini player bar + controls

**Files:** `AudioPlayerBar.tsx`, `SpeedControl.tsx`, `ReciterSelector.tsx` (word-sync badge), `RepeatControls.tsx`, `AudioDockSpacer.tsx`

Fixed bottom bar (hidden when idle); loading/buffering/error+retry; elapsed time in isolated 1 Hz child; speeds 0.75×–1.5×; repeat single ayah / range ×2/3/5/10/∞.

### Phase 5 — Quran Radio

**Files:** `src/app/radio/page.tsx`, `RadioPanel.tsx`, `Navbar.tsx` (Radio link)

`mode: "radio"` on the shared player — not a second system. `ended` swaps `src` synchronously; prefetches next chapter JSON on last verse; wraps 114 → 1; skips at most one broken chapter then stops.

### Phase 6 — Documentation & QA

**Files:** `docs/audio-sources.md`, `docs/plan.md` §3 register, `docs/regression-checklist.md` (M2 section)

Live-domain manual pass still required for audible playback, iOS Safari, and full M1+M2 checklist.

## Failure modes & degradation


| Scenario                                      | Detection                | Behaviour                                                                                |
| --------------------------------------------- | ------------------------ | ---------------------------------------------------------------------------------------- |
| Audio JSON fetch fails                        | fetch rejects            | Error state + inline retry in bar; cache evicted; reader fully usable                    |
| MP3 fails mid-playback                        | `error` event            | Error + retry near same verse; radio auto-skips once, stops after 2 consecutive failures |
| Buffering / slow network                      | `waiting` event          | Spinner on bar and pressed button                                                        |
| Segments missing for reciter                  | checked at load          | Effective `hasWordTiming: false` → ayah-level highlight only                             |
| Malformed/float/overlapping segments          | `sanitizeTimings`        | Dropped/normalized before use; never throws                                              |
| Segment matches no rendered word              | equality miss            | Nothing highlights; previous word kept on gaps                                           |
| `word.audio_url` null                         | `getWordAudioUrl`        | Speaker hidden; click no-op                                                              |
| iOS autoplay policy                           | by construction          | Playback from gesture only; queue advance reuses unlocked element                        |
| Interruption (call, background tab)           | `pause` event            | State → paused; resume needs fresh gesture                                               |
| Rapid play/reciter switch / navigate mid-play | load-token + play guards | Stale loads discarded; navigation does not stop audio                                    |
| Corrupt `rq-audio-settings`                   | migration guard          | Defaults (Alafasy, 1×)                                                                   |
| Unknown persisted reciter id                  | `getReciter` fallback    | Default reciter, no crash                                                                |

## Non-regression guarantees

- All new props optional with defaults = previous behaviour; idle DOM matches M1 (dock `h-0`, bar renders nothing).
- Touch tap-word → meaning popover unchanged; desktop hover tooltip unchanged.
- No changes to `quranApi.ts`, `types/quran.ts`, `ReaderSettingsContext`, `rq-reader-settings`, fonts, display modes, keyboard nav, or routing.
- Highlight is background-colour only — Arabic text is never animated or reflowed.
- Idle sessions produce zero player-driven re-renders (store notifies nothing).
